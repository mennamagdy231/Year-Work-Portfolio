Python Implementation

# Linear Search — O(n)
def linear_search(arr, target):
    comparisons = 0
    for i, val in enumerate(arr):
        comparisons += 1
        if val == target:
            return i, comparisons
    return -1, comparisons

# Binary Search — O(log n)
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    comparisons = 0
    while lo <= hi:
        mid = (lo + hi) // 2
        comparisons += 1
        if arr[mid] == target:
            return mid, comparisons
        elif arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1, comparisons

  • → No preprocessing needed
  • → Works on any data structure

Disadvantages

  • → Slow for large datasets
  • → Checks every element unnecessarily

  • → Highly efficient for large data
  • → Predictable performance

Disadvantages

  • → Sorting adds O(n log n) overhead
  • → Not suited for linked lists
TASK 02

Greedy — Job Sequencing with Deadlines

Problem Explanation

Given n jobs, each with a deadline and a profit, schedule at most one job per time slot to maximise total profit. Each job takes exactly 1 unit of time. A job j can only be scheduled if it is completed by its deadline.

Algorithm Steps

1. Sort jobs by profit (descending)

Greedy choice: always prefer the highest-paying available job first.

2. Find max deadline → create slot array

Array of size max_deadline tracks which time slots are filled.

3. For each job (in sorted order)

Find the latest free slot ≤ job's deadline. Assign it there.

4. Collect scheduled jobs & sum profits

Unscheduled jobs are discarded — not profitable enough given deadlines.

Python Implementation

def job_sequencing(jobs):
    # jobs = [(id, deadline, profit), ...]
    jobs.sort(key=lambda x: x[2], reverse=True)
    max_d = max(j[1] for j in jobs)
    slots = [None] * (max_d + 1)
    result, total = [], 0

    for jid, deadline, profit in jobs:
        for s in range(min(max_d, deadline), 0, -1):
            if slots[s] is None:
                slots[s] = jid
                result.append((s, jid, profit))
                total += profit
                break

    result.sort()  # sort by slot
    return result, total

# Sample input
jobs = [
    ('A', 2, 100), ('B', 1, 19),
    ('C', 2, 27), ('D', 1, 25),
    ('E', 3, 15)
]
schedule, profit = job_sequencing(jobs)
# Output: [A, C, E] → profit = 142

Why Greedy Works Here — Proof Sketch

At each step we pick the highest-profit unscheduled job that still fits a free slot. If we swapped it with a lower-profit job, total profit could only decrease. The exchange argument guarantees global optimality when profits are independent and each job occupies exactly one slot.

Python Implementation

def lcs(X, Y):
    m, n = len(X), len(Y)
    dp = [[0]*(n+1) for _ in range(m+1)]

    for i in range(1, m+1):
        for j in range(1, n+1):
            if X[i-1] == Y[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j],
                               dp[i][j-1])

    # Traceback
    lcs_str, i, j = "", m, n
    while i > 0 and j > 0:
        if X[i-1] == Y[j-1]:
            lcs_str = X[i-1] + lcs_str
            i -= 1; j -= 1
        elif dp[i-1][j] > dp[i][j-1]:
            i -= 1
        else:
            j -= 1
    return dp[m][n], lcs_str, dp

Reflection & Analysis

Implementation notes

The DP table avoids recomputing overlapping subproblems. Each cell dp[i][j] depends only on the cell above, left, and diagonally above-left.

Complexity
Time: O(m×n) Space: O(m×n) Traceback: O(m+n)
Optimisation

Space can be reduced to O(min(m,n)) by keeping only two rows of the DP table at a time (Hirschberg's algorithm).

Real-world applications
  • → Git diff — comparing file versions
  • → Bioinformatics — DNA/protein alignment
  • → Spell checkers — edit distance basis
  • → Plagiarism detection — text similarity

Python — BFS & DFS

from collections import deque

def bfs(graph, start):
    visited, order = set(), []
    queue = deque([start])
    visited.add(start)
    while queue:
        node = queue.popleft()
        order.append(node)
        for nbr in graph[node]:
            if nbr not in visited:
                visited.add(nbr)
                queue.append(nbr)
    return order

def dfs(graph, start, visited=None):
    if visited is None: visited = set()
    visited.add(start)
    order = [start]
    for nbr in graph[start]:
        if nbr not in visited:
            order += dfs(graph, nbr, visited)
    return order

def connected_components(graph):
    visited, count = set(), 0
    for node in graph:
        if node not in visited:
            dfs(graph, node, visited)
            count += 1
    return count

Connected vs Disconnected

A connected graph has a path between every pair of vertices. A disconnected graph has two or more isolated components. Running BFS/DFS from one node won't visit nodes in other components — we must restart from an unvisited node.

# Parallel (shared memory)
from multiprocessing import Pool
import numpy as np

def partial_sum(chunk):
    return sum(x**2 for x in chunk)

data = list(range(10_000_000))
chunks = np.array_split(data, 4)

with Pool(4) as pool:
    results = pool.map(partial_sum, chunks)
total = sum(results)
