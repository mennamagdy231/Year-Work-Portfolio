Python Implementation

def lcs(X, Y):
    m, n = len(X), len(Y)

    # Build DP table — O(m×n) time/space
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i-1] == Y[j-1]:
                # Characters match: extend previous LCS
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                # No match: take best of above or left
                dp[i][j] = max(dp[i-1][j],
                               dp[i][j-1])

    # Traceback to reconstruct the LCS
    lcs_str = ""
    i, j = m, n
    while i > 0 and j > 0:
        if X[i-1] == Y[j-1]:
            lcs_str = X[i-1] + lcs_str
            i -= 1
            j -= 1
        elif dp[i-1][j] > dp[i][j-1]:
            i -= 1
        else:
            j -= 1

    return dp[m][n], lcs_str, dp

Reflection

Implementation process

The DP table was built bottom-up: each cell dp[i][j] depends on three already-computed neighbours — above dp[i-1][j], left dp[i][j-1], and diagonal dp[i-1][j-1]. This avoids redundant recursion.

Challenges faced

The main difficulty was the traceback step — knowing when to go diagonal vs. up or left. An off-by-one in index mapping (1-indexed dp vs 0-indexed strings) is a common bug to watch for.

Complexity analysis
Time: O(m × n) Space: O(m × n) Traceback: O(m + n)
Optimisation possibilities

Space can be reduced to O(min(m, n)) by storing only two rows of the table at once (rolling array). For reconstruction without the full table, Hirschberg's algorithm achieves O(min(m,n)) space in O(mn) time.

Real-world applications
  • → Git diff — comparing file versions line by line
  • → DNA/protein alignment — bioinformatics sequence matching
  • → Spell checkers — basis for edit distance (Levenshtein)
  • → Plagiarism detection — measuring text similarity
  • → Unix diff utility — outputs file change sets

The answer is stored in dp[m][n] — the bottom-right cell. The traceback follows diagonal moves (matching characters) back to the origin to reconstruct the actual subsequence string.

Space-Optimised Version

def lcs_space_optimised(X, Y):
    # Only keep 2 rows → O(min(m,n)) space
    m, n = len(X), len(Y)
    if m < n:
        X, Y = Y, X  # ensure Y is shorter
        m, n = n, m

    prev = [0] * (n + 1)
    curr = [0] * (n + 1)

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i-1] == Y[j-1]:
                curr[j] = prev[j-1] + 1
            else:
                curr[j] = max(prev[j], curr[j-1])
        prev, curr = curr, [0]*(n+1)

    return prev[n]  # length only

# Note: reconstruction requires Hirschberg's algorithm
# for O(n) space while preserving traceback ability.

This version uses O(n) space instead of O(m×n), making it practical for very long strings. Trade-off: you lose the ability to reconstruct the sequence without the full table.
