Connected & Disconnected Graphs

Graph traversal with DFS and BFS — visualised & explained

Graph Types — Explanation

Connected Graph

A graph is connected if there exists a path between every pair of vertices. Starting BFS or DFS from any single node will visit all nodes in the graph. There is exactly one connected component.

Disconnected Graph

A graph is disconnected if at least one pair of vertices has no path between them. It consists of two or more isolated components. Traversal from one node cannot reach nodes in other components — the algorithm must be restarted for each unvisited node.

Python — BFS & DFS

from collections import deque

# ── BFS (Breadth-First Search) ──────
def bfs(graph, start):
    visited = set([start])
    queue   = deque([start])
    order   = []
    while queue:
        node = queue.popleft()   # FIFO
        order.append(node)
        for nbr in graph[node]:
            if nbr not in visited:
                visited.add(nbr)
                queue.append(nbr)
    return order


# ── DFS (Depth-First Search) ────────
def dfs(graph, start, visited=None):
    if visited is None:
        visited = set()
    visited.add(start)
    order = [start]
    for nbr in graph[start]:
        if nbr not in visited:
            order += dfs(graph, nbr, visited)
    return order


# ── Count connected components ───────
def count_components(graph):
    visited, count = set(), 0
    for node in graph:
        if node not in visited:
            dfs(graph, node, visited)
            count += 1
    return count

Real-World Applications

BFS Use Cases

  • → GPS navigation — shortest route (unweighted)
  • → Social networks — "People you may know" (degree of separation)
  • → Web crawlers — crawl pages level by level
  • → Peer-to-peer networks — finding nearest peers
  • → Broadcasting — network packet flooding

DFS Use Cases

  • → Topological sorting — task dependency ordering
  • → Maze solving — exploring all paths
  • → Compiler design — detecting cycles in dependency graphs
  • → Connected components — island counting in grids
  • → Strongly connected components — Kosaraju's / Tarjan's algorithm
