TASK 02

Greedy Algorithm

Job Sequencing with Deadlines — maximise profit by scheduling optimally

Problem Explanation

Given n jobs, each with a deadline and a profit, schedule at most one job per time slot to maximise total profit. Each job takes exactly 1 unit of time. A job can only earn its profit if it is completed on or before its deadline. The greedy choice is to always prefer the highest-paying available job that can still be scheduled.

Algorithm Steps

Step 1 — Sort jobs by profit (descending)

Greedy principle: always try to schedule the most profitable unscheduled job first.

Step 2 — Find max deadline → create time-slot array

Initialise an array of size max_deadline. Each index represents one time unit. All slots start empty.

Step 3 — For each job (highest profit first)

Scan backward from the job's deadline to slot 1. Place the job in the latest available slot that is ≤ its deadline. This keeps earlier slots free for later jobs.

Step 4 — Collect scheduled jobs & sum profits

Jobs that couldn't be placed are discarded. Report the schedule and total profit.

Python Implementation

def job_sequencing(jobs):
    # jobs = [(id, deadline, profit), ...]

    # Step 1: Sort by profit descending
    jobs.sort(key=lambda x: x[2], reverse=True)

    # Step 2: Create slot array
    max_d = max(j[1] for j in jobs)
    slots = [None] * (max_d + 1)
    result, total = [], 0

    # Step 3: Schedule each job
    for jid, deadline, profit in jobs:
        for s in range(min(max_d, deadline), 0, -1):
            if slots[s] is None:
                slots[s] = jid
                result.append((s, jid, profit))
                total += profit
                break

    result.sort()  # sort by time slot
    return result, total


# ── Sample Input ───────────────────
jobs = [
    ('A', 2, 100),
    ('B', 1, 19),
    ('C', 2, 27),
    ('D', 1, 25),
    ('E', 3, 15),
]

schedule, profit = job_sequencing(jobs)

# Slot 2: C ($27)

Why the Greedy Strategy is Optimal — Proof Sketch

Suppose the greedy algorithm picks job G over job X at some step. If we swap and schedule X instead, total profit decreases (since profit(G) ≥ profit(X) by construction). This exchange argument proves that no other ordering can yield a higher profit, given the constraints.

The key insight: placing each job in the latest possible slot before its deadline keeps earlier slots available for future jobs, maximising flexibility.
