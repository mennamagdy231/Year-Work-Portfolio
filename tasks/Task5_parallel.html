Architecture, execution model, advantages, limitations & real-world applications

Core Concepts

Parallel Algorithms

Run on a single machine with multiple cores or processors that share the same memory. Tasks are split into sub-tasks that execute simultaneously using threads or processes. Communication happens through shared variables in RAM — nanosecond latency.

Shared Memory Multi-core CPU / GPU OpenMP / pthreads

Distributed Algorithms

Run on multiple independent machines connected over a network. Each node has its own memory. Coordination is achieved through message passing (MPI, RPC). Communication adds network latency but enables near-unlimited horizontal scaling.

Message Passing Multiple Machines MPI / MapReduce

Python Implementation Examples

Parallel — Shared Memory (multiprocessing)

from multiprocessing import Pool
import numpy as np

def compute_chunk(chunk):
    # Each process works on its slice
    return sum(x**2 for x in chunk)

def parallel_sum_of_squares(data, workers=4):
    chunks = np.array_split(data, workers)
    with Pool(workers) as pool:
        # All workers run simultaneously
        results = pool.map(compute_chunk, chunks)
    return sum(results)

data = list(range(10_000_000))
total = parallel_sum_of_squares(data)

# With 4 cores: ~4× speedup (ideal)
# Shared memory → no network cost

Distributed — Message Passing (MPI)

from mpi4py import MPI

comm = MPI.COMM_WORLD
rank = comm.Get_rank()   # node ID
size = comm.Get_size()   # total nodes

# Master node prepares data
data = None

# Gather results at master

Amdahl's Law — Theoretical Speedup Limit

Amdahl's Law states that the maximum speedup from parallelism is limited by the sequential fraction of the program:

# Amdahl's Law
# S = speedup, p = parallel fraction, n = processors

def amdahls_speedup(p, n):
    # p = 0.9 means 90% of work is parallelisable
    return 1 / ((1 - p) + p / n)

for n in [1, 2, 4, 8, 16, 1000]:
    s = amdahls_speedup(p=0.9, n=n)
    print(f"n={n:5d}: speedup = {s:.2f}×")

# n=    1: speedup = 1.00×
# n=    2: speedup = 1.82×
# n=    4: speedup = 3.08×
# n=    8: speedup = 4.71×
# n=   16: speedup = 6.40×
# n= 1000: speedup = 9.91× (max ≈ 10×)
# Even with infinite processors, max = 1/(1-p) = 10×

This is why distributed systems pursue different tasks per machine (microservices, MapReduce) rather than just parallelising one task — to escape Amdahl's ceiling.

Real-World Applications

Parallel Algorithms in Practice

  • → GPU rendering — thousands of shader cores working in parallel
  • → Video encoding — frames processed simultaneously
  • → Scientific simulation — physics, fluid dynamics on HPC clusters
  • → Machine learning — matrix operations parallelised via CUDA
  • → Parallel merge sort — divide array, sort halves concurrently

Distributed Algorithms in Practice

  • → Google Search — MapReduce across thousands of servers
  • → Blockchain — consensus across globally distributed nodes
  • → Apache Spark — big data analytics on clusters
  • → CDNs — geographically distributed content delivery
  • → Distributed databases — Cassandra, DynamoDB, CockroachDB
