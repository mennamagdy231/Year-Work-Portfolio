Python Implementation

# ── Linear Search ── O(n) ──────────
def linear_search(arr, target):
    comparisons = 0
    for i, val in enumerate(arr):
        comparisons += 1
        if val == target:
            return i, comparisons
    return -1, comparisons


# ── Binary Search ── O(log n) ───────
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    comparisons = 0
    while lo <= hi:
        mid = (lo + hi) // 2
        comparisons += 1
        if arr[mid] == target:
            return mid, comparisons
        elif arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1, comparisons


# ── Sample run ─────────────────────
arr = sorted([3,7,15,22,37,41,58,63])
target = 37

li, lc = linear_search(arr, target)
bi, bc = binary_search(arr, target)

For n = 1 million, binary search needs only log₂(1,000,000) ≈ 20 comparisons — while linear search may need all one million.

Linear Search

✅ Advantages

  • → Works on unsorted arrays — no pre-processing needed
  • → Simple to understand and implement in any language
  • → Works on any data structure (arrays, linked lists, etc.)
  • → Efficient for very small datasets
  • → Can find all occurrences of the target in one pass

❌ Disadvantages

  • → Slow for large datasets — checks every element unnecessarily
  • → O(n) worst case makes it unusable for big data
  • → No benefit from data being sorted

Binary Search

✅ Advantages

  • → Logarithmic time — extremely fast even at n = 1 billion
  • → Very efficient for large, sorted datasets
  • → Predictable and consistent performance
  • → Used in databases, OS file systems, dictionaries

❌ Disadvantages

  • → Requires array to be sorted beforehand
  • → Sorting adds O(n log n) overhead if data is unsorted
  • → Only works on random-access structures (arrays), not linked lists
  • → More complex to implement (off-by-one errors common)

Execution Time & Efficiency Summary

Both algorithms achieve O(1) space complexity. The key difference is execution time as input grows. Linear search's time grows proportionally with n, while binary search's time grows with the logarithm of n — making it dramatically faster for large inputs.

However, if the array is unsorted and a sort is needed first, the sorting cost (O(n log n)) must be factored in. Linear search wins for one-time searches on unsorted data; binary search wins for repeated searches on sorted data.
